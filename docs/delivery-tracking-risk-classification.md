# 배송 위험 분류 & 정체 판정 (배송추적 → 슬랙)

작성일: 2026-06-28

배송추적 탭에서 미완료 송장의 우체국 이벤트를 보고 **위험 건을 분류**해 표에 색칠하고,
하루 1통 슬랙 다이제스트로 알린다. 분류·정체 판정에 들어가는 도메인 규칙을 여기 기록한다.
구현: `easy-fulfill.py` — `_classify_risk()`, `_business_elapsed_hours()`, `_collect_risk_rows()`,
`_populate_tracking_table()`.

## 위험 분류 (`_classify_risk`)

완료(`완료여부=Y`)이거나 **정체 아님**이면 위험 아님(None). 정체로 판정된 미완료 건만 분류한다.

| 분류 | 조건 | 의미 | 색 |
|------|------|------|----|
| (제외) | 상태 ∈ `배달준비` | 곧 배달 → 정상 | — |
| **수거누락** | 상태 `운송장출력` | 출력만 하고 수거 안 됨(다음날까지 출고 X) | 주황 |
| **허브정체** | 위치에 허브 키워드 포함 | 물류센터·집중국에서 정지 → **분실·사고 의심(최우선)** | 빨강 |
| **이동정체** | 그 외 | 이동 중 멈춤 | 주황 |

우선순위(다이제스트 정렬): 허브정체 → 수거누락 → 이동정체.

### 허브 키워드 (`_RISK_HUB_KEYWORDS`)
`("물류", "허브", "터미널", "집중국", "교환센터")` — 위치 문자열에 **부분일치**.

- **중요**: `우편집중국`·`국제우편교환센터`는 실제 우체국의 핵심 허브다. 초기 버전엔
  `물류/허브/터미널`만 있어 "제주우편집중국" 같은 위치가 **이동정체로 오분류**됐다(가장
  위험한 허브정체를 놓침). `집중국`·`교환센터`를 추가해 바로잡았다(2026-06-28).
- 키워드 추가/수정 시 이 표와 상수를 함께 갱신할 것.

## 정체 판정 — 영업일 기준 (`_business_elapsed_hours`)

정체 여부 = **마지막 이벤트시각(없으면 감지시각) 이후 경과가 `stale_hours` 초과**.
단, 경과 시간은 **달력 시간이 아니라 영업일(월~금) 기준**으로 환산한다.

### 왜 영업일 기준인가
우체국은 **토·일에 배송물을 움직이지 않는다.** 달력 시간으로 재면 금요일 저녁 도착 건이
월요일 아침엔 약 60시간 "무이동"으로 잡혀 **월요일마다 위험 오탐이 폭증**한다. 그래서
구간 중 토·일에 걸친 시간을 빼고 평일 구간만 누적한다.

- 예) 금 18:00 → 월 09:00: 금 18~24(6h) + 월 0~9(9h) = **15h** (토·일 48h 제외).
- 알림 자체는 이미 주말 미발송이지만(`_is_weekday`), 그것만으론 **월요일에 밀려 한꺼번에
  터지는 오탐**을 못 막는다. 정체 시간 계산을 영업일 기준으로 바꿔야 근본 해결.
- **공휴일은 아직 미반영** — 설/추석 등 평일 공휴일엔 여전히 과대평가된다. 추후 공휴일
  캘린더 도입 여지(TODO).

### 기준 시간(`stale_hours`) — 사용자 설정 가능
- 기본 **12시간(영업)**. **설정 팝업 "정체기준(시간)" 스핀박스**로 1~168 범위 조절.
- 공유 「설정」 탭 `stale_hours` 에 저장돼 **전원 적용**(다른 PC도 다음 동기화에 반영).
- 다이제스트 문구도 "기준 평일 N시간 무이동"으로 표기해 영업일 기준임을 명시.

## 알림 발송 (요약)
- **하루 1통 다이제스트** — 「새로고침」(택배사 API 조회) 완료 시에만 검토(`_maybe_send_daily_digest`).
- **평일만** 발송(`_is_weekday`), 같은 위험 목록이면 서명 비교로 재발송 안 함, 발송일은 공유
  「설정」 탭으로 중복 방지. 자세한 슬랙 게이트는 [inquiry-alerts-naver-coupang.md] 참고.
