# 11번가(11st) 주문 처리 지원

## 배경

기존 앱은 "파일 불러오기"로 네이버/쿠팡/지마켓 주문 엑셀을 읽어 마크다운 주문목록을 만들고, 그 데이터로 우체국 송장 양식(엑셀)을 출력한다. 여기에 **11번가(11st)** 채널을 추가했다.

핵심 차별점: **11번가는 Google 스프레드시트 탭이 없다.** 따라서 네이버/쿠팡과 달리 상품코드 매핑·상품 링크가 불가능하고, 일괄발송도 사용하지 않는다. xls에서 추출 가능한 정보만 마크다운으로 만들고 송장 양식만 출력하는 컨셉이다. 이는 이미 동일한 방식으로 동작하던 **지마켓(gmarket) 처리 로직**과 사실상 같아서, 지마켓 코드를 템플릿으로 복제·수정해 구현했다.

## 동작 사양

### 입력 파일
- 형식: `.xls` (OLE2/BIFF). `pandas.read_excel(path, header=2)` 로 읽음 (xlrd 필요).
- **헤더는 3행**(0-based index 2). 1·2행은 비어있음.
- 파일명 패턴: `숫자_logistics.xls`, `숫자_logistics (n).xls`
  - 정규식: `^\d+_logistics(?: \(\d+\))?\.xls$`
  - 예: `73043880_logistics.xls`, `73043880_logistics (1).xls`

### 컬럼 매핑 (헤더명 정확 매칭)
| 용도 | 헤더명 | 비고 |
|------|--------|------|
| 주문번호 | `주문번호` | 그룹 키 |
| 수취인명 | `수취인` | 마크다운/송장 통합 키 |
| 상품명 | `상품명` | |
| 옵션 | `옵션` | 비어있으면 '없음' |
| 수량 | `수량` | |
| 금액 | `주문금액` | 선택(없으면 0). 만원 표기에 사용 |
| 휴대폰 | `휴대폰번호` | 송장 "수취인 이동통신" |
| 전화번호 | `전화번호` | 송장 "수취인 전화번호" |
| 우편번호 | `우편번호` | 송장 출력 시 5자리 zfill |
| 주소 | `주소` | 송장 "수취인 주소" |
| 배송메시지 | `배송메시지` | 선택. 송장 "배송메세지" |

### 결정사항 (사용자 확정)
- **상품코드 표기**: 마크다운 상품코드 자리는 `[PASS]` (지마켓과 동일). 상품코드 매핑 없음.
- **금액 기준**: `주문금액`(L열)만 합산. 배송비는 합산하지 않음.
- **순번**: 지마켓처럼 일자별 누적. `database/order_index.json` 에 `11st` 키로 로컬 저장.
  단, **스프레드시트로 푸시하지 않음**(11st는 시트 미연동). `update_11st_index()` 와
  `on_11st_index_changed()` 는 `save_index_values(push_sheet=False)` 로 호출한다.

### 마크다운 출력 예시
```
[ ] 1.한상용 - 2.1만
▶ [PASS]**[ 1 개 ]** - 18650 배터리 2S1P 7.4V 3350mAh 케이블 타입 ( 옵션 : 없음 )

[ ] 2.유지선(서산중앙고) - 1.5만
▶ [PASS]**[ 1 개 ]** - ESP32 DevkitC V4 최신 개발 보드 ESP-NOW 지원 ( 옵션 : 없음 )
```

### 제외 기능
- **일괄발송**: `generate_invoice_file()` 은 naver/coupang만 분기. 11st는 추가하지 않음(미사용).
- **상품코드/상품 링크**: 스프레드시트 미연동으로 사용 불가.

## 검수 체크리스트 (내일 GUI 확인)
1. [ ] "불러오기" → `73043880_logistics.xls` 선택 시 마크다운이 출력되고 클립보드에 복사되는가.
   - 위 예시처럼 `[ ] N.수취인 - X.X만` / `▶ [PASS]...` 형식.
   - 참고: 격리 테스트(파싱 로직)에서는 수취인 3명(한상용/유지선/신지은) 정상 출력 확인됨.
2. [ ] 환경설정 탭에 "11번가" 순번 입력칸이 보이고, 불러오기 후 값이 증가하는가.
3. [ ] 앱 재실행 후에도 `order_index.json` 의 `11st` 값이 유지되는가.
   `git`/스프레드시트 시트에는 11st 가 push 되지 않는가.
4. [ ] 송장 양식 생성 → `output/하이제니스 폼_*.xlsx` 에 수취인/주소/전화/휴대폰/우편번호/상품명이 올바른가.
5. [ ] 기존 네이버/쿠팡/지마켓 불러오기·송장·인덱스 동작이 깨지지 않았는가(회귀).

## 알려진 가정 / 추가 확인 필요
- 파일명 선행 숫자(`73043880`)가 판매자/계정 식별자라는 가정. 다른 계정의 11번가 다운로드 파일도
  `_logistics.xls` 로 끝나는지 실제 샘플로 재확인 권장.
- 헤더가 항상 3행에 위치한다는 가정. 다른 다운로드 파일에서도 동일한지 확인 권장.
