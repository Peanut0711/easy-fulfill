## 빠른 종합 테스트 (5단계)

순서대로 한 번씩 돌려보면 3개 커밋을 모두 커버합니다.

1. **앱 실행** → 로딩 카드 없이 즉시 메인 화면이 뜨고 바로 클릭/작업 가능 → #1
2. **주문 엑셀 불러오기** → "불러오는 중…" 스피너가 멈추지 않고 매끄럽게 돌고 MD 미리보기 정상 → #3
3. **송장/주문 처리로 인덱스 증가** → 화면 끊김 없음 + 구글 시트에 반영 → #2
4. **인덱스 새로고침 버튼** → 다이얼로그 표시 + 값이 시트와 일치 → #3
5. **앱 켜둔 채 5분 이상 대기** (스크롤/입력하며) → 주기적 멈칫 없음 → #2

## #1 — 앱 시작 (커밋 `5097b7f`)

- [ ] 앱 실행 시 **"초기 데이터를 불러오는 중… 99%" 전체화면 카드가 더 이상 뜨지 않는다.**
- [ ] 실행 직후 **바로 메인 화면**이 보이고, 동기화 완료 전에도 버튼 클릭·탭 이동이 된다.
- [ ] 인덱스 동기화 라벨이 잠깐 **"Google 시트와 인덱스 동기화 중…"** → 곧
      **"스프레드시트와 동기화됨 (마지막 반영 HH:MM:SS)"** 로 바뀐다.
- [ ] 시작 후 네이버·쿠팡·지마켓 인덱스 값이 시트의 오늘 날짜 값과 일치한다.
- [ ] (체감) 예전보다 동기화가 빨리 끝난다.

## #2 — 폴링(2.5분)·인덱스 쓰기 (커밋 `0826b4c`)

**폴링**
- [ ] 앱을 켜둔 채 5분 이상 두면서 스크롤/입력해도 **2.5분마다 화면이 멈칫하지 않는다.**
- [ ] 다른 기기/사람이 시트의 오늘 인덱스를 바꾸면, 최대 2.5분 안에 앱 인덱스에 자동 반영된다.
- [ ] 인덱스 입력칸에 **포커스를 둔 동안**에는 폴링이 값을 덮어쓰지 않는다.

**쓰기**
- [ ] 송장/주문 처리로 인덱스가 올라갈 때 **화면이 끊기지 않는다.**
- [ ] 처리 직후(약 1초 내) 구글 시트 「일별 주문번호」 탭 오늘 행의 네이버·쿠팡·지마켓 값이 갱신된다.
- [ ] 인덱스를 연속으로 여러 번 바꿔도 **마지막 값이 시트에 정확히** 반영된다(중간값 유실 없음).
- [ ] 네트워크가 끊긴 상태에서 인덱스를 바꾸면 라벨에 "저장 대기…" 류 안내가 뜨고,
      연결 복구 후 다음 변경/폴링 때 반영된다(앱이 멈추지 않는다).

## #3 — 파일 불러오기·수동 새로고침 (커밋 `968ce6e`)

**파일 불러오기**
- [ ] 네이버 주문 엑셀 불러오기 → "주문 정보를 불러오는 중…" **스피너가 끊김 없이 돌고**,
      완료 후 주문 MD 미리보기가 정상 생성된다.
- [ ] 쿠팡 주문 엑셀도 동일하게 정상 처리된다(상품코드/VP 링크 포함).
- [ ] 지마켓 주문 엑셀도 정상 처리된다.
- [ ] 불러오기 시 인덱스가 시트 최신값으로 맞춰진 뒤 번호가 매겨진다.
- [ ] 잘못된 파일명을 고르면 기존처럼 "잘못된 파일명" 경고가 뜬다.

**수동 새로고침 버튼**
- [ ] 새로고침 클릭 → 처리 오버레이 후 **"스프레드시트에서 인덱스를 불러왔습니다."** 다이얼로그가 뜬다.
- [ ] 시트에 오늘 행이 없을 때 클릭하면 **"…각 1로 새로 만들었습니다."** 다이얼로그가 뜨고
      시트에 오늘 행이 생성된다.
- [ ] 로그인 만료/오류 시 경고 다이얼로그가 뜨고 앱이 멈추지 않는다.

## 알려진 무관한 기존 이슈 (이번 변경과 무관 / 참고용)

- 콘솔 인코딩이 cp949일 때 시작 시 로그에
  `'cp949' codec can't encode character '✓'` → "인덱스 로드 실패, 기본값(1)" 메시지가 보일 수 있음.
  이는 `print("✓ …")`의 ✓ 문자 인코딩 문제로, **기능에는 영향 없음**(시트 동기화가 곧 올바른 값으로 덮어씀).
  거슬리면 별도로 정리 가능.
